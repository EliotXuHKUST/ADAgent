二、各层职责说明
1️⃣ 数据采集层
•	TV / 机顶盒行为日志
•	内容与广告元数据
•	用户账号与设备信息
特点：高吞吐、实时/离线并存，不使用大模型
 
2️⃣ 特征与规则层
•	用户活跃度、偏好、消费能力
•	内容类型、受众、风险等级
•	运营规则、广告合规、频次控制
特点：稳定、可解释、可控
 
3️⃣ 语义建模层（LLM 核心使用点）
•	User Modeling Prompt → 用户语义模型
•	Content Modeling Prompt → 内容语义模型
输出： - 统一的语义标签与抽象描述 - 可缓存、可版本化
 
4️⃣ AI TV Agent 决策层
•	User Insight Agent：理解用户当前状态
•	Strategy Agent：生成营销/推荐策略意图
•	Review Agent：效果复盘与策略反馈
特点： - 大模型做“思考”，不做“执行” - 多 Agent 职责隔离
 
5️⃣ 终端呈现层
•	内容与广告组合展示
•	开机 / 首页 / 播放前贴等场景
特点：低延迟、稳定渲染
 
三、总体设计原则
•	大模型只用于 语义建模与策略生成
•	推荐、广告、排期由传统系统执行
•	在线轻决策，离线重建模
•	所有关键输出结构化、可回溯
 
四、一句话总结
这是一个“数据驱动 + 大模型语义建模 + 多 Agent 决策”的智能电视增长操作系统架构。

六、3000 QPS 显卡与成本估算（CFO 版本）
核心假设
•	LLM 实时介入率：1–3%
•	主模型：Qwen2.5-32B
•	推理框架：vLLM
显卡配置
模块	数量	说明
主 LLM 推理	8–10	承载 60 QPS 实时推理
冗余 / 灰度	2	SLA / AB Test
Embedding	2	向量召回
合计	12–14 张	稳定商用
成本量级（私有化）
•	单卡 A100 80GB：≈ 10–15 万人民币
•	GPU 一次性投入：≈ 150–200 万
对比：传统“全请求 LLM”方案 ≥ 1000 万
 
七、对外一句话话术（给 TV 厂商 / 运营商）
我们不是用大模型“直接替代推荐系统”，而是让大模型只负责高价值决策。 在 3000 QPS 规模下，真正的大模型实时推理不到 3%， 因此整套系统只需要十几张显卡即可私有化部署， 同时满足稳定性、合规与成本要求。
 
八、关键护城河（非 PPT 话术）
•	Agent 决策结果可缓存、可复用
•	用户 / 内容语义模型可长期复利
•	模型可替换，策略资产不可复制
本质：这是一个“AI 决策操作系统”，不是模型 Demo。

第 16 页｜为什么能支撑 3000 QPS（缓存设计）

•	Agent 决策高度可复用
•	缓存的是“决策意图”，不是用户请求
 
第 17 页｜四级缓存命中机制（核心）
•	L0 规则短路：无需决策直接返回
•	L1 策略模板缓存：固定场景策略
•	L2 语义簇缓存：用户 × 内容语义
•	L3 Prompt 结果缓存：兜底抗峰值
TV 场景天然适合高缓存命中
 
第 18 页｜L2 语义簇缓存（关键护城河）
•	用户行为 → 语义向量 → 聚类
•	内容元数据 → 内容语义簇
•	缓存 Key：
o	用户语义簇 × 内容语义簇 × 场景
缓存对象：结构化策略意图
 
第 19 页｜真实命中率拆解（示例）
层级	命中率
L0 规则短路	50%
L1 策略模板	25%
L2 语义簇	20%
L3 Prompt	3%

第 20 页｜为什么缓存是安全且可控的
•	决策结果可版本化
•	策略可灰度 / 回滚
•	不缓存用户隐私原始数据
缓存的是“策略”，不是“个人”
